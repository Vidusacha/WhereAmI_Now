# Дорожная карта проекта WhereAmI (V2)

Эта дорожная карта описывает основные этапы превращения прототипа в масштабируемую ИИ-платформу, готовую к продакшену.

## Фаза 1: Фундамент и Ядро 🛠️
* **Цель:** Создать надежную и масштабируемую бэкенд-инфраструктуру.
* **Основные задачи:**
  * Развернуть локальную архитектуру Kubernetes (K8s) (Traefik, Postgres).
  * Создать бэкенд на базе FastAPI.
  * Настроить новую схему базы данных (SQLAlchemy) с поддержкой системы модерации (`pending_ai_proposal`).

## Фаза 2: ИИ-Пайплайн "Discovery" 🤖
* **Цель:** Автоматизировать сбор и обработку политических данных с максимальной экономией токенов (Cascade Strategy).
* **Основные задачи:**
  * Внедрить **Механизм Каскада (Fallback Cascade)**: API Википедии/Кнессета -> Прямое скачивание PDF (PyMuPDF) -> Базовый Scraper (BeautifulSoup) -> Tavily `/extract` (только для сайтов с анти-бот защитой).
  * Разработать модуль "Поиск осей" (ИИ читает аналитику и предлагает новые оси конфликта).
  * Разработать модуль "Поиск партий" (строгое исключение министерств, используя открытые API).
  * Внедрить **Ручное добавление статических ссылок** (Static Source URL) для загрузки прямых источников (например, манифестов или баз данных).
  * Настроить локальное офлайн-хранилище для скачанных документов (с готовностью к переносу в AWS S3).

## Фаза 3: Кроссплатформенный Фронтенд (Flutter) 📱
* **Цель:** Создать единый пользовательский интерфейс для Веба и Мобильных устройств.
* **Основные задачи:**
  * Инициализировать проект на Flutter.
  * Создать панель администратора (для проверки и одобрения предложенных ИИ партий, осей и вопросов).
  * Переписать публичный интерфейс опросника "Где я на политической карте".

## Фаза 4: Обратный поиск и Генерация вопросов 🔍
* **Цель:** Углубить понимание ИИ политического ландшафта (через Google PSE).
* **Основные задачи:**
  * Внедрить "Обратный поиск": ИИ генерирует точные запросы, **Google PSE** находит Trust-ссылки (топ 5), механизм Каскада извлекает текст в Markdown.
  * ИИ оценивает позиции партий на основе чистого текста.
  * Автоматизировать создание вопросов для пользователей на основе найденных осей.

## Фаза 5: Релиз и Масштабирование 🚀
* **Цель:** Запуск в продакшен.
* **Основные задачи:**
  * Перенести офлайн-хранилище файлов в облако (AWS S3 / MinIO).
  * Развернуть кластер Kubernetes на реальном облачном хостинге (DigitalOcean, AWS, GCP).
  * Запустить Web-версию.
  * Подготовить сборку Android-приложения (.apk) из той же самой кодовой базы Flutter.
