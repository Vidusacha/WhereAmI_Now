# Пошаговый план разработки (V2)

Этот документ содержит детальные технические шаги по переходу на архитектуру V2.

## Шаг 1: Инициализация БД и Миграции
1. Настроить `backend/database.py` для асинхронного подключения SQLAlchemy к PostgreSQL.
2. Инициализировать **Alembic** (`alembic init alembic`) для управления версиями базы данных.
3. Сгенерировать первую миграцию на основе созданных моделей (`axes`, `parties`, `questions`, `party_scores`, `scraped_documents`).
4. Применить миграции к локальному контейнеру Postgres.

## Шаг 2: Создание CRUD-эндпоинтов на FastAPI
1. Создать `backend/api/routes/parties.py` для обработки `GET /parties`, `POST /parties` (для ручного добавления) и `PUT /parties/{id}/approve` (одобрение ИИ-предложений).
2. Создать `backend/api/routes/axes.py` для управления осями.
3. Создать `backend/api/routes/sources.py` для обработки `POST /sources`, куда будут отправляться **прямые (Static) ссылки** для парсинга (например, документы gov.il).
4. Подключить эти роуты в `main.py`.
5. Протестировать API через встроенный Swagger UI (`/docs`).

## Шаг 3: ИИ-Воркеры и Каскадный Скрейпер (Fallback Cascade)
1. Создать `backend/services/scraper_service.py` и реализовать **Механизм Каскада**:
   - Если URL это PDF -> скачать и прочитать через `PyMuPDF`.
   - Если это обычный HTML -> скачать через `requests` и распарсить через `BeautifulSoup`.
   - Если сайт защищен (Cloudflare/403) -> отправить 1 токен в `Tavily` `/extract` для получения Markdown.
2. Создать `backend/services/ai_service.py` для общения с локальной Ollama LLM.
3. Написать строгие промпты: *"Извлекай ТОЛЬКО политические партии или фракции. СТРОГО ИГНОРИРУЙ государственные учреждения, министерства, военные ведомства."*
4. Настроить фоновые задачи (через `BackgroundTasks` в FastAPI), чтобы парсинг не тормозил ответы API.

## Шаг 4: Реализация "Обратного поиска" (Reverse Axis Search)
1. Создать функцию `generate_search_queries(axis_description)` в `ai_service.py`.
2. Интегрировать **Google Programmable Search Engine (PSE)**. Отправлять туда запросы от ИИ, получать топ-5 точных (Trust) URL-адресов.
3. Передать эти ссылки в `scraper_service.py` (Каскад) для извлечения полного текста статей.
4. ИИ читает текст и выставляет позиции партий по оси (от -1 до 1), сохраняя результат в базу данных.

## Шаг 5: Админ-панель на Flutter
1. Запустить `flutter create frontend` для создания каркаса приложения.
2. Настроить роутинг (например, `go_router`).
3. Создать страницу `/admin` с вкладками: **Ожидающие Оси**, **Ожидающие Партии**, **Ожидающие Вопросы**.
4. Сделать таблицы, которые загружают данные из FastAPI и отображают кнопки "Approve" (Одобрить) / "Reject" (Отклонить).

## Шаг 6: Публичный опросник на Flutter
1. Переписать логику вычислений из Streamlit на язык Dart.
2. Создать экраны прохождения опросника.
3. Настроить расчет дистанции между ответами пользователя и позициями партий (либо на стороне клиента, либо через эндпоинт `/calculate` в API).

## Шаг 7: Тестирование в локальном Kubernetes
1. Убедиться, что `docker-compose.yml` прокидывает файлы кода для мгновенной перезагрузки (live-reload).
2. Собрать Docker-образы для Backend и Frontend.
3. Применить конфигурации `k8s/` к локальному кластеру (Docker Desktop K8s или Minikube), чтобы убедиться, что Traefik правильно перенаправляет трафик `/api` на Python, а остальной — на Flutter.
