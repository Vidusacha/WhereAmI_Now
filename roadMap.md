# Project WhereAmI_Now: Antigravity 2.0 (AGY) Roadmap
## Фаза 1: Развертывание инфраструктуры (База данных и LLM)
*   **Инициализация SQLite:** Создание локальной базы данных `whereami_core.db` внутри рабочей среды AGY.
*   **Схема базы данных (полностью динамическая):**
    *   Таблица `prompts_library`: хранение шаблонов промптов (system, user). Позволяет менять логику LLM без изменения кода.
    *   Таблица `axes_dictionary`: хранение выявленных водоразделов (ID, полюса).
    *   Таблица `parties_registry`: динамический список политических партий (добавляются LLM при парсинге).
    *   Таблица `vector_snapshots`: хранение координат партий с привязкой к `snapshot_id` и дате.
*   **Подключение LM Studio:** Настройка локального API-клиента на порт `1234` для взаимодействия с моделью Mistral.

## Фаза 2: Модуль динамического обнаружения (Daily Discovery Pipeline)
*   **Настройка Cron/AGY Scheduler:** Автоматический запуск скрипта сбора данных раз в сутки.
*   **Пайплайн агрегации:**
    *   Парсинг RSS-лент и новостных агрегаторов.
    *   Прогон ивритоязычного текста через локальный модуль машинного перевода (перевод на английский язык требуется для корректной работы Mistral).
*   **Prompt 1 (Axis Discovery):** Отправка переведенного текста в LLM с промптом из БД для выявления новых политических или социальных осей. 
*   **Запись осей:** Вставка новых осей в `axes_dictionary` со статусом `pending` (до ручной или автоматической валидации).

## Фаза 3: Модуль скоринга и актуализации (Temporal Tracking)
*   **Prompt 2 (Party Identification):** Запрос к LLM для выявления актуальных партий из собранного текста (учет расколов и слияний партий).
*   **Prompt 3 (Vector Scoring):** Динамическая сборка промпта. Скрипт извлекает все активные оси из `axes_dictionary`, формирует JSON-инструкцию и требует от LLM проставить веса для каждой партии на основе текстов платформ и заявлений.
*   **Генерация Снапшота:** Сохранение новых весов в таблицу `vector_snapshots`. Каждое изменение формирует новую временную точку для расчета динамики политического ландшафта.

## Фаза 4: Генератор анкеты (Dynamic UI & Questionnaire)
*   **Prompt 4 (Questionnaire Generation):** LLM получает список актуальных осей из БД и генерирует формулировки вопросов и варианты ответов (от -1.0 до 1.0) для пользователя. Ни один вопрос не зашит в код интерфейса.
*   **Интерфейс Streamlit:** Скрипт читает сгенерированные вопросы из БД и динамически отрисовывает элементы управления (слайдеры, радиокнопки).
*   **Расчет расстояний (Математическое ядро):** Вычисление Евклидова расстояния между вектором ответов пользователя и векторами партий в последнем снапшоте. Применение PCA (Principal Component Analysis) для сжатия $N$-мерного пространства в 2D-график.
*   **Рендеринг результатов:** Вывод позиции пользователя на графике и списка наиболее релевантных партий.
