# Project WhereAmI_Now: Antigravity 2.0 (AGY) Roadmap
**Core Philosophy:** Zero Hardcode. Symmetric Measurement. Temporal Tracking.

## Фаза 1: Инфраструктура и Версионированная База Данных (SQLite + LM Studio)
*   **Инициализация AGY Environment:** Настройка Python-окружения и коннекторов к локальному API (Mistral на порту `1234`).
*   **Реляционная схема с поддержкой версионирования:**
    *   `prompts_library`: Хранение всех системных промптов (ID, `version`, текст, дата). Изменение логики LLM генерирует новую версию, а не затирает старую.
    *   `axes_dictionary`: Динамический список социокультурных водоразделов (ID, полюс -1.0, полюс +1.0, статус).
    *   `parties_registry`: Актуальный список партий (напр., Likud, Yesh Atid, The Democrats, Shas) и метаданные.
    *   `dynamic_questionnaires`: Сгенерированные наборы вопросов, привязанные к осям. Каждая генерация получает уникальный `questionnaire_version`.
    *   `party_simulations`: Неизменяемая таблица (append-only) ответов LLM от лица партий (`snapshot_id`, `party_id`, `question_id`, `score`, `justification_quote`).
    *   `user_sessions`: Неизменяемая таблица ответов пользователя (`session_id`, `questionnaire_version`, `question_id`, `score`).

## Фаза 2: Модуль Daily Discovery (Парсинг и Детекция)
*   **AGY Cron Job (Ежедневный запуск):**
    *   **Сбор данных:** Агрегация RSS-лент (Кан 11, Ynet, Haaretz), заявлений политиков и свежих электоральных опросов.
    *   **Машинный перевод:** Локальный перевод ивритоязычного контента на английский для стабильной работы базовой Mistral.
*   **Prompt 1 (Axis & Party Discovery):** Динамический промпт анализирует сводку за 24 часа. Цель: найти новые водоразделы (например, новые законопроекты о призыве, вопросы бюджета 2026) или новые политические альянсы. 
*   **Запись в БД:** Новые оси помечаются как `pending_review`. Новые заявления партий складируются в `party_documents` с текущей датой.

## Фаза 3: Генерация Опросника (Questionnaire Generation)
*   **Триггер:** Запускается при появлении новых подтвержденных осей или при устаревании прошлой анкеты.
*   **Prompt 2 (Question Generator):** LLM получает список активных осей из `axes_dictionary` и генерирует список утверждений.
    *   *Пример поведения:* Для оси "Религия и Государство" LLM генерирует: *"Общественный транспорт должен функционировать по субботам в светских муниципалитетах"*.
*   **Запись:** Сохранение сгенерированного JSON-массива вопросов в таблицу `dynamic_questionnaires` с присвоением новой версии (`v.1.x`).

## Фаза 4: Симуляция Партий (Simulated Testing)
*   **Идея:** Партии и пользователь оцениваются одним и тем же "измерительным прибором".
*   **Prompt 3 (Persona Testing):** LLM инициируется в роли лидера конкретной партии (Persona Prompting).
    *   *Входные данные:* Актуальный `questionnaire_version` и тексты заявлений/платформы партии из БД.
    *   *Задача LLM:* Ответить на каждый вопрос анкеты от -1.0 до +1.0 *строго* на основе загруженных документов. Для каждого ответа LLM обязана выдать цитату-обоснование (`justification_quote`).
*   **Snapshots:** Результаты сохраняются в `party_simulations`. Если партия изменила свою позицию с прошлого снапшота (математическая дельта > 0.1), система маркирует это как "Смещение платформы" (Party Drift).

## Фаза 5: Пользовательское Тестирование и UI (Flutter Dashboards)
*   **Дэшборд 1: Анкета (The Test):** Flutter динамически отрисовывает слайдеры (-1.0 до +1.0) на основе активного `questionnaire_version`.
*   **Дэшборд 2: Результаты (Where Am I?):**
    *   Расчет Евклидова расстояния между вектором ответов пользователя и векторами партий (из последнего снапшота Этапа 4).
    *   Вывод топ-3 наиболее подходящих партий.
    *   *Обоснование модели:* При клике на совпавшую партию UI показывает цитаты-обоснования, сгенерированные LLM на Этапе 4.
*   **Дэшборд 3: Монитор трендов (Temporal Analytics):**
    *   Сравнение текущей `user_session` с историческими сессиями. UI подсвечивает: *"Ваша позиция по судебной системе сместилась на +0.3 вправо по сравнению с 2025 годом"*.
    *   Радары смещения партий: Визуализация того, как конкретная партия меняла риторику перед выборами.
